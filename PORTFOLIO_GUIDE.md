# Web Engineer Portfolio Site - カスタマイズガイド

このポートフォリオサイトは、エンジニア×デザイン思考をアピールするための転職用ポートフォリオです。
以下のセクションで、カスタマイズ方法を説明します。

## 📋 サイト構成

### 1. ナビゲーション（Navigation）
- **ファイル**: `client/src/pages/Home.tsx` の `Navigation()` コンポーネント
- **カスタマイズ**: ロゴテキストやリンク先を変更可能

### 2. ファーストビュー（Hero Section）
- **内容**: キャッチコピー、自己紹介、CTA（Call To Action）ボタン
- **カスタマイズポイント**:
  - `<h1>Web Engineer</h1>` → あなたの肩書きに変更
  - サブタイトル → あなたの専門分野に変更
  - 説明文 → あなたの価値提案に変更

### 3. About セクション
- **内容**: プロフィール、スキル、強み
- **カスタマイズポイント**:
  - プロフィール画像（現在はプレースホルダー）
  - スキルリスト → あなたの実際のスキルに変更
  - 強みの説明 → あなたの実績に基づいて記述

### 4. Philosophy セクション
- **内容**: 設計思想、ユーザー中心の考え方、学習姿勢
- **カスタマイズポイント**:
  - 3つの哲学セクション → あなたの価値観に合わせて変更
  - アイコン → lucide-react から選択可能

### 5. Works セクション
- **内容**: プロジェクト紹介（エンジニア作品 + デザイン作品）
- **カスタマイズポイント**:
  - `projects` 配列に実際のプロジェクトを追加
  - 各プロジェクトの `title`, `description`, `tags`, `link` を更新
  - プレースホルダー画像の代わりに実際の画像を使用

### 6. Skills セクション
- **内容**: 技術スタック（Backend / Frontend / Design / Tools）
- **カスタマイズポイント**:
  - `skills` オブジェクトに実際のスキルを追加

### 7. Contact セクション
- **内容**: 連絡先（メール、GitHub、LinkedIn）
- **カスタマイズポイント**:
  - メールアドレス → あなたのメールアドレスに変更
  - GitHub URL → あなたのGitHubプロフィールに変更
  - LinkedIn URL → あなたのLinkedInプロフィールに変更

## 🎨 デザインカスタマイズ

### カラーパレット
- **ファイル**: `client/src/index.css`
- **主要色**:
  - `--primary: #1E40AF` （深い青 - 信頼感と知性）
  - `--background: #FFFFFF` （純白）
  - `--foreground: #1A1A1A` （濃いスレート）
  - `--secondary: #F3F4F6` （ライトグレー）

### タイポグラフィ
- **見出し**: Playfair Display（セリフ、優雅）
- **本文**: Inter（サンセリフ、読みやすい）
- **フォント設定**: `client/index.html` の Google Fonts リンク

### アニメーション
- **ライブラリ**: Framer Motion
- **設定**: `client/src/pages/Home.tsx` の `itemVariants`, `containerVariants`
- **カスタマイズ**: `transition` プロパティで速度や遅延を調整可能

---

## 🖼️ 画像の追加

### プロフィール画像
1. 画像を `client/public/` に保存（例: `profile.jpg`）
2. `Home.tsx` の About セクションで以下のように変更:
```jsx
<div className="bg-secondary rounded-lg aspect-square mb-6">
  <img src="/profile.jpg" alt="Profile" className="w-full h-full object-cover rounded-lg" />
</div>
```

### プロジェクト画像
1. 画像を `client/public/` に保存（例: `project1.png`）
2. `ProjectCard` コンポーネントの `image` プロップを画像パスに変更
3. または、`<img>` タグを使用して実際の画像を表示

## 🚀 デプロイ

### Manus での公開
1. サイトのカスタマイズが完了したら、Management UI の「Publish」ボタンをクリック
2. カスタムドメインを設定可能（Settings → Domains）

### 外部ホスティングへのエクスポート
1. `pnpm build` でビルド
2. `dist/` フォルダを任意のホスティングサービスにアップロード

## 📝 コンテンツ作成のポイント

### エンジニア作品紹介
- **課題**: 何を解決したのか？
- **技術**: 使用した言語・フレームワーク
- **UI/UX**: ユーザー体験向上のための工夫
- **成果**: 実装した結果、学んだこと

### デザイン作品紹介
- **コンセプト**: 誰に、何を伝えたかったのか？
- **デザイン意図**: 色・形・フォントの選定理由
- **プロセス**: 制作過程での工夫

### Philosophy セクション
- **技術力と美学の融合**: エンジニアとしての論理性 + デザイナーとしての感覚
- **ユーザー中心の設計**: なぜこの機能が必要か、ユーザーの課題は何か
- **継続的な学習**: UI/UX領域への関心、スキルの拡張

## 🔧 開発コマンド

```bash
# 開発サーバー起動
pnpm dev

# ビルド
pnpm build

# プレビュー
pnpm preview

# TypeScript チェック
pnpm check

# フォーマット
pnpm format
```
